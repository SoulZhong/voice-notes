# Windows v0.5.0 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Document, build, verify, and publish official Windows x64 installers on the existing GitHub `v0.5.0` release.

**Architecture:** Keep the published `v0.5.0` tag immutable while building Windows artifacts from the latest PR #57 commit. Treat README accuracy, local build verification, checksums, GitHub upload, and post-upload inspection as separate release gates.

**Tech Stack:** Markdown, Vitest, SvelteKit, Tauri 2, Rust, PowerShell, GitHub CLI

## Global Constraints

- Keep `package.json`, `src-tauri/Cargo.toml`, and `src-tauri/tauri.conf.json` at version `0.5.0`.
- Never move, recreate, or force-update the existing `v0.5.0` tag.
- Never delete or replace existing macOS release assets.
- Build from the latest pushed commit on `codex/windows-icon-unification`.
- Publish exactly `voice-notes_0.5.0_x64-setup.exe`, `voice-notes_0.5.0_x64_en-US.msi`, and `SHA256SUMS-windows.txt`.
- Do not instruct users to disable Windows Security, SmartScreen, or Smart App Control globally.
