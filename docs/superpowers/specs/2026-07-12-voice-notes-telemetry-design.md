# voice-notes 产品遥测（每日使用统计）设计

日期：2026-07-12
状态：已定稿（用户确认）

## 背景与目标

应用将分发给多用户，需要了解「每天有多少人在用、用了哪些功能」。选型结论（用户三轮确认）：

- **产品遥测**（多用户匿名上报），不是本地个人统计页。
- 服务端用**第三方平台 Aptabase**（隐私优先、官方 Tauri 2 插件、可自部署），不自建。
- 隐私默认值：**默认开启 + 设置页可关**，首启明示告知。

## 指标

Aptabase 看板开箱提供，无需自己开发展示端：

- 日活/周活/月活（按匿名会话自动统计，不存设备标识）。
- 版本分布、OS 分布（插件自动附带）。
- 下述各事件的每日计数与属性分布。

## 事件清单（首批 6 个）

| 事件 | 属性 | 说明 |
|---|---|---|
| `app_started` | — | 启动即发，撑起日活 |
| `recording_started` | `source`: `mic` / `system` / `both` | 核心功能使用 |
| `recording_stopped` | `duration_bucket`: `<5m` / `5-30m` / `30-60m` / `>1h` | 时长只报桶，不报精确值 |
| `note_refined` | `provider`: 预设类别（如 `doubao` / `openai` / `custom`） | 精修使用率；只报预设名 |
| `note_exported` | `format`: `md` | 导出使用率 |
| `mcp_tool_used` | `tool`: 工具名（固定枚举） | MCP/Agent 接入面使用 |

次要事件（删除笔记、说话人合并等）首批不埋，要用再加。

### 属性规范（硬约束・隐私红线）

属性值只允许**固定枚举**与**数值桶**。以下内容一律禁止上报：会议内容、笔记标题、说话人名、文件路径、API key、模型接入点 ID、任何自由文本。红线写进 telemetry 模块文档注释，并有防回归测试。

## 架构

### Rust 端统一埋点

- 接入 `tauri-plugin-aptabase`（官方 Tauri 2 插件），lib.rs 注册，App-Key 配置一行。
- 埋点全部放在 Rust 命令层/pipeline 层，**前端不埋**：录制、精修、导出、MCP 调用都汇聚在 Rust 侧；前端埋点会漏掉 MCP/CLI 触发的使用。

### telemetry 薄封装

新增 `telemetry` 模块，对外唯一入口 `track(app, event)`：

- 内部读设置开关，关闭即短路（无需重启）。
- 事件与属性用 Rust enum 建模（`TelemetryEvent` 枚举携带各自的枚举属性），编译期杜绝自由字符串进属性。
- 转成插件的 `track_event(name, props)` 调用。

### 失败处理

- 插件自带批量缓冲与重试；网络不可达（含大陆访问 EU/US 端点不通）**静默丢弃**。
- 上报绝不影响主流程：不阻塞、不弹错、不记用户可见日志。
- 将来量大或合规需要时可自部署 Aptabase（App-Key 换 `A-SH-*` + 自定义 host），客户端只换配置。

## 隐私开关与告知

- `Settings` 增加 `telemetry_enabled: bool`，`#[serde(default = "default_true")]`，与现有字段模式一致。
- 设置页加开关行，文案：「匿名使用统计：仅上报功能使用次数与版本信息，绝不包含任何会议内容。」
- 首启 WelcomeOverlay 加一句告知 + 指向设置页。
- 关闭立即生效。

## 测试

- 开关关闭时 `track` 不产生任何上报调用（对插件调用打桩/注入验证）。
- 事件属性由枚举建模，编译期保证无自由文本；加一条防回归测试锁住属性序列化形状。
- 不做网络集成测试（上报可靠性是插件职责）。

## 不做的事（YAGNI）

- 不自建服务端；不做本地统计页；不做漏斗/留存/用户级分析。
- 不采集设备标识、不做用户 ID 关联。
- 首批不埋次要事件。
